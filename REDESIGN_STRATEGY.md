# InfoLady.online — Стратегический план редизайна UX/UI

## 📊 Статус анализа

### Выявленные критические проблемы

#### 🔴 ФЕЙКОВЫЕ И ДЕМО ДАННЫЕ (КРИТИЧНО)
```
❌ "120+ проверок уже сделано" - неизвестно реально ли это
❌ "7 ошибок находим у каждого второго сайта" - статистика
❌ "+40% рост заявок" - результаты, нет доказательств
❌ Красное предупреждение "7 ошибок находим" - звучит как демо
❌ Карточки тарифов с зарезервированными местами
❌ Случайные цифры везде
```

**Решение:** Убрать все фейковые метрики, заменить на честные (или скрыть)

#### 🔴 CTA КОНФЛИКТЫ
```
❌ Header CTA: "Получить данный аудит сайта"
❌ Top bar: "Узнать, где теряете клиентов"
❌ Hero input: "Проверить мой сайт бесплатно"
❌ Mobile-cta: "Получить клиентов"
❌ Telegram бот ссылки везде
❌ Нет главного CTA - все равноправны
```

**Решение:** 1 главный CTA на каждой странице, остальные вторичные

#### 🔴 UX/UI ПРОБЛЕМЫ
```
❌ Визуальная иерархия нарушена
❌ Перекрытия на мобильных (mobile-cta перекрывает контент)
❌ Отступы и spacing не единообразны
❌ Формы выглядят как V1, не premium
❌ Placeholder'ы с техническими текстами
❌ Меню не функционирует
❌ Нет логики воронки: вход → анализ → результат → платёж
```

#### 🔴 КОПИРАЙТИНГ
```
❌ Шаблонные фразы ("узнайте за 30 секунд")
❌ Нет конкретики: что именно получает пользователь?
❌ Нет ясной разницы между бесплатным и платным
❌ Нет пояснений про Telegram бот
❌ Нет пояснений про PDF (откуда берется, когда придет)
❌ Нет real success stories
```

#### 🔴 ДОВЕРИЕ & БЕЗОПАСНОСТЬ
```
❌ Нет социального доказательства (отзывы, кейсы)
❌ Нет информации о компании
❌ Нет ясности: это SaaS? Агентство? Скам?
❌ Telegram бот кажется отдельным сервисом
❌ Нет политик конфиденциальности на главной
❌ Фейковые "успехи" снижают доверие
```

## 📑 ДОКУМЕНТЫ ПО ПРОБЛЕМАМ

### Проблема 1: Фейковые данные в Hero

**Текущее:**
```html
"120+ проверок уже сделано"
"7 ошибок находим у каждого второго сайта"
"+40% рост заявок после правок"
```

**Проблема:** Это звучит как демо-данные. Нет источников.

**Решение:**
- ВАРИАНТ A: Если данные есть - вставить реальные
- ВАРИАНТ B: Если данных нет - переформулировать:
  ```
  "Проверим ваш сайт на 50+ параметров"
  "Покажем конкретные проблемы, которые теряют вам клиентов"
  "Вы получите действительный план роста за 24–48 ч"
  ```

### Проблема 2: CTA конфликты

**Примеры конфликтов:**
1. Top bar: "Узнать где теряете" (стоит всё время)
2. Header: "Получить аудит"
3. Hero: "Проверить мой сайт"
4. Footer: "Получить клиентов"
5. Mobile-cta: "Get Clients"

**Решение:** ОДНА главная CTA - "Проверить сайт бесплатно"

### Проблема 3: Мобильные перекрытия

**Issue:** Mobile-cta закрывает контент внизу страницы

**Решение:** 
- Убедиться что body имеет padding-bottom (есть ✓)
- Проверить что все input'ы имеют правильный z-index
- Для final CTA при прокрутке - использовать sticky

### Проблема 4: Типизация форм

**Issue:** `<input data-placeholder-en="..." data-placeholder-ru="..."/>`

**Решение:** Нормальные HTML5 placeholders:
```html
<input 
  type="email" 
  placeholder="you@company.com"
  aria-label="Email address"
/>
```

### Проблема 5: Telegram не объяснен

**Текущее:** Просто ссылка на бот везде

**Решение:** 
1. Объяснить ЧТО делает бот
2. КОГДА пользователь к нему переходит
3. ЧТО там происходит
4. КОГДА он получает результаты

Примерно:
```
👇 Нажмите кнопку ниже

Бот InfoLady
↓
Отправит вам быстрый анализ
↓
Покажет главные 2–3 проблемы
↓
Предложит полный отчет
↓
Вы получите PDF за 24–48 часов
```
